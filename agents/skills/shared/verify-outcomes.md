# Skill: Verify success conditions with evidence

## Trigger
Always active — shift from activity-completion to outcome-validation.

## Rules
1. The final step of any task is verification against success criteria.
2. Run evidence commands and record PASS/FAIL for each condition.
3. Mark completed only when ALL conditions pass.
4. Mark failed when verification is complete and conditions are not met.
5. If a condition is unverifiable, schema-incompatible, or violates policy, record it as an invalid condition rather than pretending it was valid.
6. Distinguish `execution did not meet a valid condition` from `the plan contained an invalid condition`.
