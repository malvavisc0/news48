How it works

1

Find

news48 pulls stories from selected feeds, keeps an eye on source quality, and fetches the full article even when a site makes that annoying.

2

Clean

It turns messy source pages into clean, readable article pages with summaries, topics, tags, and less noise.

3

Check

Important claims are pulled out and checked against live web evidence, then labeled as verified, disputed, mixed, or still unclear.

4

Publish, then vanish

Stories appear with source links, fact-check badges, and grouped topics. Then the whole thing forgets them on purpose after 48 hours.

Why people like it

Less sludge

Similar stories are clustered together, so you get fewer duplicate headlines pretending to be different news.

More signal

Source links stay visible, summaries stay short, and claim checks make the page more useful than a simple repost.

Short memory

The 48-hour window keeps the experience fresh. No endless archive, no stale junk pile, no pretending old news is new.

Actually useful

It is built for people who want a quick read, and for tools that want structured news through MCP.

MCP, without the mystery

news48 exposes its news data via the Model Context Protocol, so AI assistants like Claude, Cursor, and others can browse, search, and inspect stories in real time. Two connection modes are available:

Local (stdio)

The local server runs as a subprocess. It is the easiest option for desktop AI tools and gives full access to local news48 data. Add this to your MCP client config:

Claude Desktop / Cursor — mcp_config.json 
{
  "mcpServers": {
    "news48": {
      "command": "uv",
      "args": ["run", "news48-mcp"],
      "cwd": "/path/to/news48"
    }
  }
}

No auth needed here. It runs locally and can see everything, including articles that are still mid-pipeline.

Remote (Streamable HTTP)

The remote endpoint lives at /mcp. It uses a Bearer token and returns fully parsed articles, ready for tools that connect over the network.

1. Generate an API Key

CLI 
news48 mcp create-key --label "my-assistant"

This prints a key like n48-abc123.... Save it when you see it. Later, you can rotate or revoke it, but not recover the original value.

2. Configure Your MCP Client

Claude Desktop / Cursor — mcp_config.json 
{
  "mcpServers": {
    "news48": {
      "url": "https://{{ mcp_domain }}/mcp",
      "headers": {
        "Authorization": "Bearer n48-your-api-key-here"
      }
    }
  }
}

3. Manage Keys

CLI 
# List active keys
news48 mcp list-keys

# Revoke a key
news48 mcp revoke-key n48-your-api-key-here

Available Tools

Both local and remote endpoints expose the same useful set of tools:

get_briefing Top stories, trending topics, breaking news
search_news Full-text search with filters
get_article Full article with fact-check claims
list_categories All categories with article counts
list_clusters Topic clusters with grouped stories
get_stats Pipeline and article statistics

Self-hosting in one breath

news48 is self-hosted, container-friendly, and built for people who like owning their stack. Bring feeds, storage, an LLM, and a little curiosity.

Quick Start 
# Clone and configure
git clone https://github.com/malvavisc0/news48.git
cd news48
cp .env.example .env   # Edit with your settings

# Start everything
docker compose up -d

# Add feeds and generate an MCP key
news48 feeds add https://feeds.bbci.co.uk/news/rss.xml
news48 mcp create-key --label "my-key"

# Open the web interface
open http://localhost:8765

The full stack includes storage, queues, search, anti-bot fetches, the web app, workers, and optional local AI horsepower. Everything is still purged after 48 hours, because that is part of the point.
